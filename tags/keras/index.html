부족한 데이터 늘리는 방법(Agumentation)

ImageDataGenerator 사용법

Augumentation 기초적 설명!!

Augumentation??
augumentation이라는 단어가 어떻게 사용되는지 먼저 예시를 들겠습니다.
A: 모델 학습하는데 augumentation은 어떤 방식으로 하셨나요?
(==데이터 증가를 위해 어떤 방법을 사용하셨나요??)
B: horizontal_flip 를 사용하고 zoom_range = 0.2 그리고 zca_whitening 를 사용했습니다 (여기에 나오는 영어 단어들은 뒤에 실습결과로 설명드리겠습니다.)

(비지도 학습) 딥러닝을 한다는 것은 우리가 갖고있는 데이터를 기반으로 새로운 데이터의 정보를 예측하는 겁니다.

이때 갖고있는 데이터를 기반으로 학습을 하기때문에 데이터는 많으면 많을 수록 모델의 성능이 좋아집니다.

지금 당장 1,000장의 이미지를 수집해야 한다고 생각해봅시다.
크롤링으로 수집 작업을 하고.. 휴.. 중복이미지를 제거, 학습에 부적합한 이미지 제거와 같은 전처리 작업을 해야합니다.

혼자서 1,000장의 이미지를 수집하는것은 쉬운일이 아닙니다. (필자 기준..)
그냥 이미지를 500장을 수집하고 복제하면 2배가 되는데요??
이렇게 작업을 하면 학습데이터에 대해 정확도는 올라가지만 새로운 데이터에 대한 정확도는 낮아지는 과대적합(overfitting) 문제가 발생해서 안됩니다.|

이렇게 이미지 수집의 한계가 있을때
data augumentation (데이터 확대) 데이터의 양을 늘리는 작업을 해야합니다.
추가적으로 새로운 이미지 수집이 아닌 기존 이미지의 변형을 통해 새로운 이미지를 만드는 작업!!

우리는 Keras 에서 만든 API로 위에서 설명한 Agumentation을 쉽게 사용할 수 있습니다.
이제 어떻게 사용하고 결과가 나오는지 알려드리겠습니다.

ImageDataGenerator (실습)

필요한 라이브러리 호출

agumentation을 하기위한 라이브러리를 호출합니다.
(이번 실습에 사용할 라이브러리 일뿐 다른 방법으로도 실습 하실 수 있습니다.)

이미지를 1개만 호출해 어떤식으로 결과가 나오닌지 확인해보겠습니다.
<사용할 이미지>
{: width=”50%” height=”50%”}

ImageDataGenerator 객체를 생성합니다.
객체의 input 내용은 Augumentation 조건으로 여러 가지를 사용할 수 있습니다.

결과 시각화

width_shift_range
width_shift_range 는 왼쪽 또는 오른쪽으로 얼마만큼 이동할지 값을 줘서 변형하는 것을 확인했습니다.
{: width=”50%” height=”50%”}

zoom_range
부분확대를 통해 데이터에 변화를 줍니다.
{: width=”50%” height=”50%”}

rotation_range
이미지를 회전시키므로 변화를 줍니다.
{: width=”50%” height=”50%”}
